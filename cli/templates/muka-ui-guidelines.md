## Importing Styles

Import styles once in your app's root layout or entry point:

```tsx
// Default theme (muka-light)
import '{{PACKAGE_NAME}}/styles';

// Or choose a specific theme:
import '{{PACKAGE_NAME}}/styles/muka-light.css';
import '{{PACKAGE_NAME}}/styles/muka-dark.css';
import '{{PACKAGE_NAME}}/styles/wireframe-light.css';
import '{{PACKAGE_NAME}}/styles/wireframe-dark.css';
```

### Theme Switching

To support dynamic theme switching, import all themes and toggle via a CSS class or attribute:

```tsx
// In your root layout
import '{{PACKAGE_NAME}}/styles/muka-light.css';
import '{{PACKAGE_NAME}}/styles/muka-dark.css';

// Apply theme via data attribute or class
<html data-theme="muka-dark">
```

## Using Design Tokens

Muka UI provides CSS custom properties (tokens) for consistent styling:

### Spacing

```css
var(--spacing-1)   /* 0.25rem */
var(--spacing-2)   /* 0.5rem */
var(--spacing-3)   /* 0.75rem */
var(--spacing-4)   /* 1rem */
var(--spacing-5)   /* 1.25rem */
var(--spacing-6)   /* 1.5rem */
var(--spacing-8)   /* 2rem */
var(--spacing-10)  /* 2.5rem */
var(--spacing-12)  /* 3rem */
```

### Colors (Semantic)

```css
/* Text */
var(--color-text-default-default)
var(--color-text-subtle-default)
var(--color-text-inverse-default)

/* Backgrounds */
var(--color-surface-level1)
var(--color-surface-level2)
var(--color-surface-level3)

/* Borders */
var(--color-border-default)
var(--color-border-contrast)
var(--color-border-focus)

/* Actions */
var(--color-action-default)
var(--color-action-hover)
var(--color-action-pressed)
```

### Typography

```css
var(--alias-font-brand-family)
var(--alias-font-brand-weight-semibold)
var(--text-label-sm-fontSize)
var(--text-label-md-fontSize)
var(--text-label-lg-fontSize)
```

## Requesting New Components

If you need a component that doesn't exist in Muka UI:

1. **Check Storybook** to confirm it's not available: {{STORYBOOK_URL}}

2. **Don't create a local workaround** — this defeats the purpose of using a design system

3. **Request the component** by:
   - Opening an issue in the muka-ui repository
   - Or building it yourself in muka-ui and submitting a PR

4. **Wait for availability** — once the component is in Storybook, you can use it

## Common Patterns

### Form with Validation

```tsx
import { Input, Button, Card } from '{{PACKAGE_NAME}}';

function LoginForm() {
  const [error, setError] = useState('');

  return (
    <Card>
      <Input
        label="Email"
        type="email"
        error={!!error}
        errorMessage={error}
      />
      <Input
        label="Password"
        type="password"
      />
      <Button variant="primary" fullWidth>
        Log in
      </Button>
    </Card>
  );
}
```

### Button Variants

```tsx
import { Button } from '{{PACKAGE_NAME}}';

// Primary action
<Button variant="primary">Save</Button>

// Secondary action
<Button variant="secondary">Cancel</Button>

// Subtle action
<Button variant="ghost">Learn more</Button>

// Sizes
<Button size="sm">Small</Button>
<Button size="md">Medium</Button>
<Button size="lg">Large</Button>
```

### Tabbed Interface

```tsx
import { Tabs, TabList, Tab, TabPanel } from '{{PACKAGE_NAME}}';

<Tabs defaultValue="tab1">
  <TabList>
    <Tab value="tab1">First</Tab>
    <Tab value="tab2">Second</Tab>
  </TabList>
  <TabPanel value="tab1">First tab content</TabPanel>
  <TabPanel value="tab2">Second tab content</TabPanel>
</Tabs>
```
