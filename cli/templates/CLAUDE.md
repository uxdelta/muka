## UI Component Constraint

**This project uses Muka UI as its sole component library. All UI must be built with components from muka-ui.**

### Rules

1. **Only use components available in Muka UI**
   - Check availability at {{STORYBOOK_URL}}
   - Import from `'{{PACKAGE_NAME}}'`

2. **No alternative styling solutions for UI components**
   - No Tailwind utilities for component styling
   - No CSS modules for UI elements
   - No styled-components or Emotion
   - No inline styles except for layout (using Muka tokens)

3. **When a component doesn't exist**
   - Do NOT create a local component as a workaround
   - Note the missing component as a blocker
   - The component must be built in muka-ui first
   - Once available in Storybook, it can be used here

### Imports

```tsx
// Import components
import { Button, Card, Input, Chip } from '{{PACKAGE_NAME}}';

// Import styles (choose one theme)
import '{{PACKAGE_NAME}}/styles';              // Default theme (muka-light)
import '{{PACKAGE_NAME}}/styles/muka-dark.css';     // Dark theme
import '{{PACKAGE_NAME}}/styles/wireframe-light.css'; // Wireframe brand
```

### Acceptable Styling Patterns

```tsx
// ✅ GOOD: Use Muka UI components
import { Button, Card } from '{{PACKAGE_NAME}}';

// ✅ GOOD: Use Muka UI tokens for layout
<div style={{
  display: 'flex',
  gap: 'var(--spacing-4)',
  padding: 'var(--spacing-6)'
}}>

// ✅ GOOD: Use semantic color tokens
<span style={{ color: 'var(--color-text-subtle-default)' }}>

// ❌ BAD: Tailwind classes
<div className="flex gap-4 p-6">

// ❌ BAD: CSS modules
import styles from './Component.module.css';

// ❌ BAD: Arbitrary values
<div style={{ color: '#6366f1', padding: '24px' }}>
```
