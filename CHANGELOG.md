# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [0.1.0] - 2026-02-25

### Added

#### Components
- **Alert** - Component for displaying important messages and notifications
- **Badge** - Component for displaying status indicators and labels
- **BottomBar** - Navigation component for bottom-aligned actions
- **Button** - Primary interactive component with multiple variants
- **Card** - Container component for grouping related content
- **Checkbox** - Form input for binary selections
- **CheckboxTile** - Tile variant of checkbox for visual selection
- **Chip** - Component for tags, filters, and compact information
- **Container** - Layout component for content containment
- **DatePicker** - Input component for date selection
- **Dialog** - Modal component for focused interactions
- **Divider** - Visual separator component
- **FormProgressBar** - Component for multi-step form progress indication
- **Icon** - Icon component with Remix Icon integration
- **Input** - Text input component for forms
- **Label** - Form label component
- **ListItem** - Component for list entries
- **PriceTag** - Specialized component for displaying prices
- **Progress** - Component for showing progress and loading states
- **Radio** - Form input for single selection from multiple options
- **RadioTile** - Tile variant of radio for visual selection
- **Section** - Layout component for content organization
- **SegmentGroup** - Component for segmented controls
- **Select** - Dropdown selection component
- **Sheet** - Slide-out panel component
- **Table** - Component for displaying tabular data
- **Tabs** - Component for organizing content in tabs
- **Tile** - Versatile container component with interactive variants
- **Toast** - Component for temporary notifications
- **Toggle** - Switch component for on/off states
- **TopBar** - Navigation component for top-aligned actions

#### Design Token System
- 4-layer token architecture (primitives, alias, semantics, components)
- Multi-brand support (Muka and Wireframe brands)
- Multi-theme support (Light and Dark themes)
- Token build system with reference resolution
- CSS custom properties generation for all token layers
- Comprehensive color, typography, spacing, and sizing tokens

#### Development Tools
- Storybook integration for component development and documentation
- Vitest test suite with unit and visual regression testing
- TypeScript support with full type definitions
- ESLint configuration for code quality
- Husky pre-commit hooks for token builds
- Build system for ESM, CJS, and TypeScript declarations

#### Automation
- GitHub Actions workflow for token building
- Automated Storybook deployment to GitHub Pages
- Token validation scripts
- Deployment diagnostic tools

#### CLI
- `muka-ui` command-line interface for project tooling

#### Documentation
- CLAUDE.md with AI coding assistant guidance
- Token system documentation
- Component architecture guidelines
- Build and development command reference

### Changed
- Initial release - no changes to document

### Fixed
- Initial release - no fixes to document

### Removed
- Initial release - nothing removed
